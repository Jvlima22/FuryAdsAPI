The three components

REST API

Node.js + Express. Receives compliance events (webhook) and serves reporting requests. No public access.

Job queue

BullMQ + Redis. Processes ad-pause actions asynchronously with idempotency (one job per tenant + ad) and exponential-backoff retries.

Google Ads adapter

The only component that calls the Google Ads API. Selects the target account (customer_id) per request, scoped to accounts under our MCC.

Authentication to Google: a single OAuth 2.0 refresh token tied to our manager account, stored as an environment secret. No end-user Google credentials are collected.
